Section 1
What is DeclareUI?

DeclareUI is a tool that turns descriptions into real, working interface code.

Here's the idea: instead of writing complex code by hand for buttons, forms, alerts, and other interface elements, you describe what you want in a simple text format called YAML. DeclareUI reads that description and converts it into professional code that works in popular frameworks like React, Vue, Angular, and others.

Think of it as a translator. You speak "description," and DeclareUI speaks "code." You focus on what you want to build. DeclareUI figures out how to build it.

It's built for every kind of builder -- whether you write code every day, you're a designer with an idea, or you're someone who just wants to create something without climbing a steep learning curve. DeclareUI makes it simpler than you think.

Section 2
YAML in 2 Minutes

YAML is just structured text with indentation -- like a grocery list with sub-items. If you've ever written a to-do list with bullet points, you already get the basic idea.

A grocery list everyday 
Fruits:
  - Apples
  - Bananas

Dairy:
  - Milk
  - Cheese

Bakery:
  - Sourdough bread
The same thing in YAML yaml 
fruits:
  - Apples
  - Bananas

dairy:
  - Milk
  - Cheese

bakery:
  - Sourdough bread

That's really it. YAML uses a few simple rules:

  • Indentation matters -- items nested under a category are indented with spaces (usually 2 spaces).
  • Colons separate keys from values -- like name: Button means "the name is Button."
  • Dashes mark list items -- like - Apples means "one item in the list."
  • Use spaces, not tabs -- YAML is picky about this one thing.

If you've ever edited a settings file, a configuration, or even a recipe in a text document, you already know YAML. It's designed to be readable by humans first, machines second.

Section 3
Tailwind in 2 Minutes

Tailwind is a way to style things using short, descriptive class names. Instead of writing traditional style code, you use simple labels that describe what you want something to look like.

It reads almost like English. Here are a few examples:

bg-blue-600 text-white px-4 py-2
Click Me
blue background, white text, horizontal & vertical padding
bg-green-500 text-white rounded-full px-6 py-3
Success
green background, fully rounded, more padding
bg-red-100 text-red-800 border border-red-300
Error: Something went wrong
light red background, dark red text, red border
bg-gray-900 text-white rounded-lg shadow-lg
Dark Card
dark background, rounded corners, shadow effect

The pattern is always: what-color-intensity. So bg-blue-600 means "background, blue, medium-dark." The number goes from 50 (very light) to 950 (very dark).

You don't need to memorize any of this. DeclareUI includes Tailwind class names in the YAML templates, and you can always tweak them by reading the class name out loud to yourself -- it usually tells you exactly what it does.

Want to go deeper? The official Tailwind documentation is excellent and full of visual examples.

Section 4
Your First Component -- Step by Step

Let's build something real. We'll create an Alert component -- those little notification boxes you see on websites. It'll take about 5 minutes.

1

Install the DeclareUI command-line tool

Open your terminal (or command prompt) and run this command. It installs the DeclareUI tool globally on your computer so you can use it from anywhere.

Terminal shell 
# Install DeclareUI globally
$ npm install -g @declareuihq/cli

Don't have npm? It comes with Node.js. Download and install Node.js first.

2

Create a new project

This creates a folder with everything set up and ready to go, including an example component to start from.

Terminal shell 
$ declareui init my-first-ui
$ cd my-first-ui
3

Open the example and understand the structure

Inside your new project, you'll find a file called button.ui.yaml. Open it in any text editor. Let's look at a simplified version and understand each part:

alert.ui.yaml source 
component: Alert

props:
  type:
    type: enum
    values: [info, success, warning, error]
    default: info
  message:
    type: string
    required: true

template:
  tag: div
  classes:
    base: "px-4 py-3 rounded-lg font-medium"
    variants:
      type:
        info: "bg-blue-100 text-blue-800"
        success: "bg-green-100 text-green-800"
        warning: "bg-yellow-100 text-yellow-800"
        error: "bg-red-100 text-red-800"
  children:
    - tag: span
      content: "$props.message"
component: Alert -- The name of your component. This becomes the name used in code (like <Alert />).
props -- These are the inputs your component accepts. Think of them as "settings." Here, you can set the type (info, success, warning, error) and the message text.
template → classes → base -- Tailwind styles that always apply: padding, rounded corners, bold text.
template → classes → variants -- Different styles depending on which type is chosen. An "info" alert is blue; an "error" alert is red.
children → content: "$props.message" -- This displays whatever text is passed as the message prop.
4

Build it

Run the build command. DeclareUI reads your YAML and generates real, production-quality code.

Terminal shell 
$ declareui build
5

See what came out

DeclareUI generated a complete React component (and Vue, Angular, etc. if configured). Here's the React output:

Alert.tsx react output 
import { forwardRef } from 'react';
import { cn } from '../utils/cn';

export interface AlertProps {
  type?: 'info' | 'success' | 'warning' | 'error';
  message: string;
}

export const Alert = forwardRef<
  HTMLDivElement, AlertProps
>(({ type = 'info', message }, ref) => {
  const classes = cn(
    'px-4 py-3 rounded-lg font-medium',
    {
      info: 'bg-blue-100 text-blue-800',
      success: 'bg-green-100 text-green-800',
      warning: 'bg-yellow-100 text-yellow-800',
      error: 'bg-red-100 text-red-800',
    }[type]
  );

  return (
    <div ref={ref} className={classes}>
      <span>{message}</span>
    </div>
  );
});
6

You just built a professional UI component!

That's it. You described what you wanted in plain, readable YAML, and DeclareUI generated production-ready code with proper TypeScript types, accessibility support, and framework-native patterns.

From here, you can change the colors, add new variants, or create entirely new components. The process is always the same: describe, build, ship.

Section 5
Using AI to Build Components

Don't want to write YAML by hand? Just describe what you need in plain language.

AI tools like Claude can generate DeclareUI components for you. You describe what you want in a conversation, and the AI writes the YAML. DeclareUI then compiles it into real code. You just describe what you want.

◈
Claude + DeclareUI via MCP
I need a notification banner that can be an info, warning, or error type. It should have a title, a message, and a dismiss button.
I'll create that for you. Let me generate the YAML component...

Created notification-banner.ui.yaml with:
• Props: type, title, message, dismissible
• Events: dismiss
• Variants: info (blue), warning (amber), error (red)

Run declareui build to compile it!

How does this work? DeclareUI has an MCP Server (Model Context Protocol). In simple terms, it's a bridge that lets AI tools talk directly to DeclareUI. The AI can look up the DeclareUI documentation, understand the YAML format, and generate valid components -- all within a conversation.

Setting it up with Claude Desktop

Add this to your Claude Desktop configuration file to enable the DeclareUI integration:

claude_desktop_config.json json 
{
  "mcpServers": {
    "declareui": {
      "command": "npx",
      "args": ["@declareuihq/mcp-server"]
    }
  }
}

Once configured, you can ask Claude to create, modify, or explain DeclareUI components. The AI writes the YAML, DeclareUI compiles the code. You just describe what you want.

Section 6
Frequently Asked Questions
Do I need to know how to code?
Not necessarily. If you can edit a text file, you can use DeclareUI. The YAML format is designed to be readable by anyone -- it looks more like a structured document than traditional programming code. That said, some familiarity with the terminal (running commands) is helpful for the build step. And if you use the AI integration, you can skip writing YAML entirely.
What is React / Vue / Angular?
These are frameworks -- toolkits that developers use to build the interactive parts of websites and apps. Think of them as different "languages" that browsers understand. Different teams prefer different frameworks. DeclareUI's superpower is that you write your component description once, and it generates code that works in any of these frameworks. So your team can use whatever they prefer.
Is this free?
Yes. DeclareUI is open source under the MIT license, which is one of the most permissive licenses in software. You can use it for personal projects, commercial products, or anything in between -- at no cost, forever.
Can I use this at my company?
Absolutely. The MIT license allows commercial use with no restrictions. Many teams use DeclareUI to maintain a consistent component library across multiple projects that use different frameworks.
What if I get stuck?
You have several options:
  • Open an issue on GitHub -- the community and maintainers are friendly and responsive.
  • Browse existing issues -- someone may have already asked your question.
  • Ask an AI assistant (like Claude) with the MCP integration -- it can help troubleshoot and explain errors.
How is this different from v0.dev or Bolt?
Tools like v0.dev and Bolt generate one-time code -- you get a snapshot that you then maintain manually. DeclareUI takes a different approach: you create reusable, testable components from a single YAML source that lives in your codebase. When you update the YAML, you rebuild and get updated code for every framework. It's a source-of-truth approach rather than a one-shot generation.
